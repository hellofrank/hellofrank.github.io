Spring注解系列一

发表于 2019-08-31 | 分类于 Spring
本文字数： 964 | 阅读时长 ≈ 2 分钟

程序员圈子里，Spring可以说是大名鼎鼎，关于Spring的知识点都可以写好几本书了。你新爷目前还没有那么本事去写任何一本关于spring的书籍，这里我只是想简单总结一下Spring常用的注解。
Spring经过这么多年的发展,功能变得越来越完善，触角越来越多，野心也越来越大，什么都想做，具体可以去看Spring官网。
你新爷刚入行的时候，spring还是2.X版本，当时做WEB开发的标准三件套还是Struct+Spring+Hibernate。
当年的三剑客现在也只有Spring活的越来越好，现在已经到5.X版本了，而且已经发展成了集团军作战。现在搞web工程，Spring MVC 基本已经成了事实标准。当年那个年代，Spring只负责类的注入。现在不一样了，现在Spring全都管了。而且使用方式也跟以前大不相同，以前Spring要配合大量的XML配置文件使用。把所有注入的类，通过XML配置来实现。现在推荐使用注解来实现。这样有个好处就是不用再写那么多配置文件了，不写配置文件能带来什么好处呢，这好处还真有，首先第一个就是当你看代码的时候不用总是在配置文件和JAVA代码之间跳来跳去了。用了注解之后，配置和代码浑然融为一体，看起来简单了，写起来也简单了。但是唯一的麻烦就是程序员有需要多学一些东西了。

好了闲话说完，开始本文的正题，新爷给大家简单介绍一下Spring的注解。

  1. @Controller。作用在类上，用于表示这是一个Controller。（真是一句废话），该注解需要配合@RequestMapping使用。这个注解下一篇回重点介绍。

  2. @Service ：作用在类上。表示业务层，具体业务逻辑写到这一层。

  3. @Repository ：作用在类上。用于表示在跟数据库相关的操作，就是常说的DAO层。

  4. @Resource ：作用在类和字段上，Java提供的注解。表示按照类的名字注入。

  5. @Autowired ：作用在类或者字段上。Spring提供的注解，按照类型注入。

  6. @Qualifier ：作用在类上，当创建多个类型相同的bean时，可以通过该注解消除混乱，配合 @Autowired 使用。

  7. @Compoent :作用在类上，通用组件注解，在不知道具体分到哪一类时使用，基本不推荐使用。
